🧠 Leo's AI 工具箱架构图 v2026.06.22

从 AI 模型到终端设备的完整工具栈架构 · 四层分层结构 · 可扩展
① 终端设备 —— 用户交互入口
📱 iPhone 主设备
飞书 — 与 OpenClaw / Codex 通讯桥接（消息通道）
Telegram — 与 Grok Build / OpenClaw 通讯桥接
Termius — 手机终端，经 Tailscale SSH 连接远程主机
② AI Agent 平台 —— 大脑 & 编排层
🛠️ Codex
AI 编码代理。接收飞书/Telegram 指令，自主编写/调试代码，管理项目文件，调用模型完成开发任务。
🤖 OpenClaw
全能 AI 数字秘书。知识查询、文件整理、邮件/日历管理、跨平台通讯、任务调度。核心入口。
⚡ Grok Build
快速原型 Agent。擅长从零到一的快速构建、创意实验、xAI 生态深度集成。
🔮 Hermes
Agent 运行时环境。管理 Node.js 包、执行脚本、维护 Agent 运行状态及插件系统。
③ 基础设施 —— 连通 & 部署层
🔁 CC Switch
API Key 鉴权代理。解决 Agent 平台对接模型时「有订阅账号、无官方 API Key」的问题。相当于智能路由 + 认证转换。
🔗 Tailscale
WireGuard 组网。设备间无公网 IP 也能直连，打通 Mac、Windows、iPhone 的安全隧道，Termius SSH 依赖它。
🐙 GitHub
代码仓库。Agent 生成的代码自动推送至远程 repo，实现版本管理、异地访问、第三方协作/调用。
☁️ Railway
云部署平台。提供虚拟服务器（云端 VPS），用于部署/运行服务端项目，一键发布。
④ AI 大模型 —— 算力 & 推理层
💬 ChatGPT · OpenAI
通用旗舰模型。文本推理、代码、多模态（视觉/语音）。通过订阅账号 + CC Switch 代理接入。
🧪 Gemini · Google
多模态大模型。长上下文、原生多模态处理。通过 Google AI API / 订阅代理接入。
🌌 Grok · xAI
实时推理模型。深度集成 X/Twitter 数据，快速推理。通过订阅账号 + CC Switch 代理接入。
🐋 DeepSeek · 深度求索
中文优化模型。高性价比推理、代码能力出色、上下文支持大。通过 API / 代理接入。
📦 Ollama 本地
本地模型部署平台。在本地 Mac 上运行开源模型（Llama、Qwen、Mistral 等），无需联网、离线可用。

📡 数据流全景

① 用户 ▸ iPhone（飞书/Telegram/Termius）发指令
② Agent 平台 ▸ OpenClaw/Codex/Grok Build 解析任务
③ 鉴权代理 ▸ CC Switch 转换订阅 → API Key
④ 模型推理 ▸ ChatGPT/Gemini/Grok/DeepSeek/Ollama
⑤ 成果落地 ▸ GitHub 存代码 / Railway 部署服务
⑥ 返回用户 ▸ 结果推送到飞书/Telegram
📋 生成工具：OpenClaw · Jenny 可扩展 · 后续可增加新工具至对应层级
